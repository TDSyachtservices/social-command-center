⚙️ Core System Logic
1. Mention Group Builder UI

Route: /mention-groups

Users can:

Create/edit/delete groups
Add members per platform
Fields per member:
platform selector
handle input
display name
LinkedIn URN field (only shown for LinkedIn)
2. Scheduled Post Composer

When creating a post:

Allow selecting platforms (Instagram, Facebook, LinkedIn)
Allow selecting mention groups
Show preview of resolved mentions per platform

Store only:

group IDs
selected platforms
3. Mention Resolver Engine

Create:

resolveMentions(postId, platform)

Behavior:
Instagram
Output: plain text @username
Must ensure exact handle formatting
Facebook
Output: @PageName or plain tagged text
Prefer page names for reliability
LinkedIn
Must prioritize:
linkedin_urn if available
fallback to @name style text
Example internal representation:
"urn:li:person:123456"
Return:
{
  finalContent: string,
  mentions: string[],
  platform: string
}
4. LinkedIn-Specific Rules (IMPORTANT)
API tagging is limited and often restricted
Always store:
linkedin_urn for people or companies
If URN exists:
prefer structured tagging
If not:
fallback to plain text @mention

Treat LinkedIn as:

“semi-structured mention system with optional enrichment”

5. Inbound Mention Detection System

Create endpoint:

/api/scan-mentions

Simulate ingestion for all platforms:

Instagram / Facebook:
Scan text for:
/@([a-zA-Z0-9._]+)/g
LinkedIn:
Scan:
@name
company/page name matches (fallback logic)

Store:

detected handles
platform
source text
post URL (if available)
