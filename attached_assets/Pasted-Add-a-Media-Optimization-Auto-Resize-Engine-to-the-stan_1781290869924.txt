Add a Media Optimization / Auto-Resize Engine to the standalone Social Command Center app.

Goal:
The app must allow a user to upload one original image or video and automatically generate the correct platform-specific media versions for Facebook, Instagram, LinkedIn, TikTok, and Website publishing.

Important:
- Keep the app standalone.
- Keep the existing Social Command Center UI and brand style.
- Do not connect live social APIs yet.
- Do not publish anything yet.
- Use mock processing or frontend simulation if backend processing is not ready.
- Build the UI and data structure so real image/video processing can be added later.
- Do not rely on social platforms to crop media correctly.
- Store the original uploaded asset and generate separate platform-ready derivatives.

Add a new module called:
Media Optimizer

Add routes/screens:
- /media-library
- /media-optimizer
- /media-optimizer/:assetId
or follow the existing routing pattern.

Core concept:
User uploads one original asset.
The app creates:
1. Original asset
2. Facebook-ready version
3. Instagram-ready version
4. LinkedIn-ready version
5. TikTok-ready version

- aspect ratio compatibility
- file type

Score labels:
- Excellent
- Good
- Needs Review
- Poor

Example:
A 4000 x 3000 image used for 1080 x 1080 should be Excellent.
A 1200 x 630 image forced into 1080 x 1920 should be Needs Review.
A 600 x 400 image used for 1920 x 1080 should be Poor.
