## Problem
The service deploys from the `main` branch with rootDirectory `/server`. Railway auto-detects `server/Dockerfile` and uses the Dockerfile builder because no `railway.json` exists on `main` to override the builder. The Dockerfile fails because its COPY commands use a `server/` prefix (e.g., `COPY server/src ./src`), but the Docker build context is already scoped to the `server/` directory — so Docker looks for `server/server/src`, which does not exist.

Build error: `Build Failed: failed to calculate checksum of ref: "/server/src": not found`

The SNAPSHOT stage succeeded. The BUILD stage failed immediately. The previous deployment (53828da9) from `main` also failed with `"/tsconfig.json": not found` — the same root cause. The last successful deployment (07e76b36) was from the `backend-foundation` branch, which has `server/railway.json` forcing `"builder": "RAILPACK"` and all required server files.

## Diagnosis
The `main` branch is missing four critical files that exist on `backend-foundation`:
1. `server/railway.json` — forces Railpack builder (bypasses Dockerfile), sets startCommand to `node dist/index.js`, preDeployCommand to `npx prisma migrate deploy`, and healthcheckPath to `/api/health`
2. `server/package.json` — declares server dependencies (express, prisma, etc.) needed for Railpack to detect Node.js and install packages
3. `server/tsconfig.json` — TypeScript compiler config needed for the build step
4. `server/package-lock.json` — npm lockfile needed for deterministic installs

Without `railway.json`, Railway sees the Dockerfile and uses it. The Dockerfile's COPY commands fail because the referenced files don't exist in the build context.

## Fix
The recommended fix is to merge the `backend-foundation` branch into `main`, which will bring all missing files at once.

Alternatively, create the following files manually:
