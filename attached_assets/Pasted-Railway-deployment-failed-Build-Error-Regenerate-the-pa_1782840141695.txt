## Problem
The Dockerfile build fails at `RUN npm run build` with `sh: prisma: not found` (exit code 127). The build script is `prisma generate && tsc --project tsconfig.json`. The `prisma` CLI is listed in devDependencies but is not available in the build container.

A fix was already pushed to HEAD changing `npm install` to `npm ci --include=dev` in the Dockerfile (server/Dockerfile), but that deployment also fails with:
```
npm error `npm ci` can only install packages when your package.json and package-lock.json or npm-shrinkwrap.json are in sync.
npm error Missing: @emnapi/core@1.11.1 from lock file
npm error Missing: @emnapi/runtime@1.11.1 from lock file
```

## Diagnosis
The `server/package-lock.json` was generated on Replit (resolved URLs point to `package-firewall.replit.local`) and is missing transitive peer dependencies of `@napi-rs/wasm-runtime`, specifically `@emnapi/core` and `@emnapi/runtime`. `npm ci` requires an exact match between `package.json` and `package-lock.json` and refuses to install when entries are missing. Without a successful install, the `prisma` CLI binary is never placed in `node_modules/.bin/`, so the build script fails.

## Fix
1. **File:** `server/package-lock.json`
2. **Action:** This file must be regenerated by running `npm install` in the `server/` directory on a local machine (not Replit). This will resolve all transitive dependencies including the missing `@emnapi/core` and `@emnapi/runtime`, and replace Replit-specific registry URLs with standard npmjs.org URLs.
3. **Steps:**
   ```bash
   cd server
   rm -rf node_modules package-lock.json
   npm install
   git add package-lock.json
   git commit -m "fix: regenerate package-lock.json for npm ci compatibility"
   git push
   ```
4. After committing the updated lockfile, `npm ci --include=dev` (already in the Dockerfile on HEAD) will find a complete, in-sync lockfile, install all dependencies including `prisma`, and the build step will succeed.
