Build a standalone frontend web app for a social media scheduling, publishing, and comment management system.

App name:
Social Command Center

Purpose:
Create a standalone admin app that will eventually connect by API to:
- the company website
- Facebook
- Instagram
- LinkedIn
- TikTok
- n8n
- local AI computer / private AI model

This is a frontend-first build. Do not implement live backend integrations yet. Use mock data, local React state, and stubbed handlers. The app should be designed so backend/API integrations can be added in later phases.

Important project clarification:
- This is NOT being embedded inside the existing public website.
- This should be a separate standalone app.
- It may later connect to the website by API.
- It may later send approved social posts to the website, blog, news feed, learning center, or project gallery.
- It may later receive website leads/comments/form inquiries by API.
- Do not modify any existing website routes, SEO pages, public navigation, schema, or marketing pages.
- Build this as its own app/dashboard.

Branding:
- Keep the visual identity consistent with the existing TDS / marine manufacturing brand.
- Use a clean, professional, premium marine-industry look.
- Avoid generic SaaS styling if it conflicts with the existing brand.
- Use refined cards, white/neutral backgrounds, strong spacing, navy/dark accents, gold or brand-accent highlights if used in the existing brand, clean tables, and professional form layouts.
- The app should feel appropriate for a 40+ year teak decking / marine manufacturing company.
- Use existing brand colors, typography, button style, border radius, shadows, hover states, and spacing if available in the project.
- If existing components are available, reuse them.
- If no component system exists, create a clean reusable component system.
